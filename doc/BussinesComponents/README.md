# Пример бизнес-компонентов любой ERP системы

При начале работы с требованиям необходимо определить компонентную модель, чтобы уже в рамках компонентной модели определять структуру каталогов будущих Feature файлов

Любая организациия и структура ее автоматизации в части бизнес приложения состоит из

* 7 основных направлений автоматизации
* для каждого из направлений выделяются 3 бизнес-компонента для будущей автоматизации
* согласно концепции декомпозиции сущностей отдельный объект 1С должен стремится к тому чтобы относится к одному бизнес-компоненту

В данном каталоге - абстрактная структура бизнес компонентов, любой современной ERP системы

~~~
  
Обратите внимание: Существуют 4 служебных каталога

Core - для фиксации низкоуровневых требований к служебным объектам 1С
Support - для фиксации требований к автоматизации службы поддержки информационной системы
Vendors - для фиксации требований к разработке встраиваемых подсистем, таких как БСП, клиент ЭДО и т.д.
Bussines - для фиксация требований к одному из 7 реальных бизнес направлений 

~~~
